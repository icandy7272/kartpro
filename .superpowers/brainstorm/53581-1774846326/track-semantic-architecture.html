第一阶段设计: 赛道语义骨架层

目标: 自动分析尽量准, 但在关键语义不确定时宁可保守提示确认, 不自信地给错建议。

三层结构
Layer 1: 遥测真相层
输入原始 GPS / VBO / 视频提取数据，产出稳定的圈、轨迹、速度、时间轴、起终线。
这里只负责“数据是什么”，不负责“应该怎么教”。
↓
Layer 2: 赛道语义骨架层
把赛道组织成 弯 - 直道 - 弯间关系 的统一模型。
这一层新增:
1. 关键出弯弯 / 长直道入口弯
2. 组合弯 / 牺牲弯 / 铺垫弯候选
3. 每个语义的置信度分数
↓
Layer 3: 教练决策层
教练建议不再直接从“单个弯速度差”出发，而是从赛道骨架推导:
这个弯是该最大化出弯，还是该为后面弯让步？
这个组合弯该看第一段还是最后一段？
自动推断流程
原始点 → 圈检测 → 弯道检测 → 直道分段
→ 赛道图结构化
→ 关键出弯弯候选
→ 组合弯 / 牺牲弯候选
→ 每个候选计算置信度
人工确认介入点
高置信度: 直接进入分析
中低置信度: 展示候选 + 一键确认/改判
允许人工确认:
1. 弯道边界 / 编号
2. 某弯是否为关键出弯弯
3. 两个相邻弯是否视为组合弯
A

结构方向对

三层结构和确认点设置都合理，可以继续细化数据模型。

B

结构基本对，但要调整

大方向可以，但确认点、语义范围或层次边界还需要改。

C

方向不对

需要换一种组织方式，不想先做这套骨架层。
